Bambu Lab Chamber Heater Automation

n8n workflow to control a chamber heater via a Tuya smart plug based on Bambu Lab printer state. Designed for ABS/ASA prints: heat when needed, cool when idle.

Download workflow JSON Setup steps

Hardware links

  • Heater (AliExpress)
  • Smart switch (AliExpress)
  • Compatible: Tuya smart plug controlling the heater

What it does

  • Listens to printer status via MQTT (LAN mode).
  • Detects high-temp materials (ABS/ASA) from bed target temp.
  • Turns heater on when chamber is cold, off when warm or idle.

Requirements

  • Self-hosted n8n with Docker or Compose.
  • Bambu Lab P1/X1 in LAN mode (or Cloud polling).
  • Tuya IoT account + smart plug.

Safety

  • Use a heater rated for enclosed spaces.
  • Do not leave unattended without thermal safeguards.
  • Verify plugs are within current limits.

Setup overview

Enable crypto in n8n so the Code node can sign Tuya requests: 
NODE_FUNCTION_ALLOW_BUILTIN=crypto

1. Tuya IoT credentials

Create a cloud project, link your app account, and note your Access ID, Access Secret, and Device ID. These are placeholders in the sample JSON.

2. MQTT access

Enable LAN mode on the printer and use the MQTT topic: device/YOUR_PRINTER_SN/report.

3. Import workflow

Import workflow.sample.json into n8n, then update the placeholders for your own device IDs and credentials.

Workflow logic

Trigger

MQTT trigger reads Bambu printer status every update.

Material check

Bed target temp >= 85C implies ABS/ASA.

Chamber check

Heat if chamber temp is below target; stop at warm threshold.

Tuya command

Signs a POST request with HMAC-SHA256 and toggles the smart plug.

Regional API endpoints

Set the Tuya endpoint for your region in the HTTP node:

US: https://openapi.tuyaus.com
EU: https://openapi.tuyaeu.com
CN: https://openapi.tuyacn.com
